01 / NOW

현재 상태

주문·결제·영수증과 지점 운영 화면의 기반이 연결돼 있습니다. 실제 운영 전에는 지점 식별과 일부 UI 기능, 외부 서비스 환경설정을 보완해야 합니다.

Customer flow

QR → Toss 결제 → 영수증 출력

QR로 CheckoutView를 열고 Toss SDK 결제 성공을 서버에서 승인한 뒤 영수증 조회·출력으로 이어집니다.

Branch operation

계정·주문·재고·매출 화면 추가

Firebase 지점 가입·로그인과 branchId 접근 검증, 지점 주문·재고·판매 화면 및 API가 포함됩니다.

03 / SCOPE

develop2.0 확장 범위

기존 주문 흐름에 더해 이 브랜치에서 실제 코드로 확인되는 주요 확장입니다.

TOSS연결

실제 Toss Payments 결제

CheckoutView가 Toss SDK 결제창을 열고 백엔드 TossPaymentGateway가 결제 승인을 처리합니다.

RECEIPT연결

영수증 조회와 로컬 출력

ReceiptController와 ReceiptService가 화면 데이터와 프린터 출력 성공·오프라인 결과를 제공합니다.

FIREBASE기반

지점 가입·로그인·승인

Firebase 세션, 지점 가입 화면, 본점 계정 승인 화면과 관련 서비스가 추가됐습니다.

ACCESS기반

지점별 데이터 접근 제어

BranchAccessService가 로그인 지점과 요청 branchId를 검증하고 비인증 접근을 차단합니다.

BRANCH기반

주문·재고·판매 관리

지점 Orders, Inventory, Sales 화면과 Controller·Service가 추가돼 운영 기능의 기본 흐름을 제공합니다.

KIOSK UI개선

1024px 키오스크 UI 적용

상품 이미지·아이콘, 핑크 테마, 4열 상품/맛 그리드와 고정 하단 액션 바가 적용됐습니다.

운영 전 남은 부분

develop2.0을 실제 매장 운영 수준으로 올리기 위해 명확히 남아 있는 항목입니다.

픽업 시간 선택

컵/콘은 있지만 지금·30분 후·1시간 후 선택은 아직 없습니다.

지점 식별

체크아웃 payload의 branchId가 임시로 1에 고정돼 있습니다.

다국어·쉬운모드 동작

버튼과 아이콘은 있으나 실제 번역·레이아웃 변경 상태 연결이 남았습니다.

추천메뉴 로직

판매량 상위 2개와 지점 선정 2개를 합치는 데이터 규칙은 확인되지 않습니다.

직원 호출 연결

bell UI는 있으나 Firebase 호출 요청과 지점 알림 수신 연결이 남았습니다.

환경설정·브랜치 정리

Toss/Firebase/프린터 운영 키 설정과 갈라진 주문 브랜치 이력 통합이 필요합니다.
