1. 배경도 (Context Diagram)

시스템의 전체적인 범위와 외부 엔티티(고객, 관리자, 결제 시스템)와의 상호작용을 정의합니다.

flowchart LR Customer([고객]) BranchAdmin([가맹점 관리자]) HQAdmin([본사 관리자]) System((아이스크림\n키오스크 시스템)) PG([결제 PG사]) Customer -- "주문/결제 정보 (장바구니)" --> System System -- "주문 결과/영수증" --> Customer BranchAdmin -- "재고 확인, 발주 요청" --> System System -- "재고/발주 현황" --> BranchAdmin HQAdmin -- "상품/분점 관리, 발주 처리" --> System System -- "시스템 운영 데이터" --> HQAdmin System -- "결제 요청 정보" --> PG PG -- "결제 승인/실패 내역" --> System
Level 0 DFD

2. 최상위 DFD (Level 1 Diagram)

ERD를 바탕으로 도출된 4개의 주요 프로세스와 데이터 저장소 간의 전체적인 흐름을 보여줍니다.
