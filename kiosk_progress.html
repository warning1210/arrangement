U

사용자

키오스크에서 상품, 맛, 옵션을 선택하고 QR 결제로 주문을 완료한다.

B

지점

주문을 제조/완료 처리하고, 재고와 상품 노출, 바쁨 상태를 관리한다.

H

본점

재고 신청 승인, 배송, 이벤트, 상품 정책, 공지와 통계를 관리한다.

전체 흐름 사용자 지점 본점
전체 구조

주문 데이터가 운영 데이터로 흐르는 구조

고객 주문이 결제에서 끝나지 않고, 재고 차감과 본점 의사결정까지 이어지는 구조입니다.

STEP 01

본점 정책 설정

상품, 가격, 이벤트, 공지, 지점 계정 기준을 관리한다.

STEP 02

지점 운영 준비

재고, 추천 메뉴, 상품 노출, 바쁨 상태를 지점 상황에 맞게 관리한다.

STEP 03

사용자 주문

매장/픽업 선택 후 상품, 옵션, 맛을 고르고 QR 결제를 진행한다.

STEP 04

주문 처리/재고 차감

지점이 주문을 완료 처리하면 판매 수량만큼 재고가 줄어든다.

STEP 05

신청/배송/통계

부족 재고를 본점에 신청하고, 누적 데이터는 통계와 이벤트 판단에 쓰인다.

Role Scenario

역할별 시나리오 전환 화면

아래 버튼으로 같은 영역에서 사용자, 지점, 본점 시나리오를 전환합니다.

User Scenario

사용자 시나리오

사용자는 키오스크에서 주문하고 결제합니다. 장바구니는 화면 상태로만 관리하고, 결제 시 주문으로 확정합니다.

U-01

먹고가기 기본 주문

광고 화면에서 시작해 QR 결제와 완료 알림까지 이어진다.

  1. 광고 화면 터치이달의 맛, 이벤트, 신제품 광고를 보고 주문을 시작한다.
  2. 먹고가기 선택주문 방식 팝업에서 매장 취식을 선택하고 상품 화면으로 이동한다.
  3. 상품과 옵션 선택사이즈를 고르고 싱글류는 컵/콘, 대용량은 스푼/드라이아이스를 선택한다.
  4. 맛 선택 완료상품별 선택 가능 개수만큼 맛을 고르면 결제 버튼이 활성화된다.
  5. QR 결제포인트/할인을 적용하고 휴대폰으로 QR코드를 스캔해 결제한다.
  6. 대기번호 발급주문 정보가 지점 주문 목록에 등록되고 완료 시 알림을 받는다.

중간 예외사항

  • 재고 0 상품은 처음부터 노출하지 않는다.
  • 맛 개수가 부족하면 결제 버튼을 비활성화한다.
  • QR 결제 실패/만료 시 QR코드를 재생성한다.
  • 주문 중 홈버튼을 누르면 취소 확인 팝업을 보여준다.
U-02

가져가기 픽업 주문

픽업 시간을 먼저 선택하고 지점은 해당 시간에 맞춰 제조한다.

  1. 가져가기 선택주문 방식 팝업에서 픽업 주문을 선택한다.
  2. 픽업 일시 선택가능한 시간대 중 픽업 예정 시간을 선택한다.
  3. 상품 구성사이즈, 옵션, 맛을 선택해 주문을 구성한다.
  4. 주문 내역 확인장바구니 화면에서 픽업 시간과 상품 구성을 확인한다.
  5. 결제 완료QR 결제 후 대기번호와 픽업 예정 시간이 표시된다.
  6. 픽업 준비지점 주문 목록에 픽업 주문으로 등록되어 시간에 맞춰 준비된다.

중간 예외사항

  • 지점이 바쁜 경우 선택 가능 시간이 뒤로 밀릴 수 있다.
  • 주문량이 많은 시간대는 비활성화한다.
  • 포장 불가 상품은 픽업 주문에서 선택을 막는다.
U-03

쉬운모드 주문

큰 글자와 단순한 그리드로 주문 부담을 줄인다.

  1. 쉬운모드 실행상품 선택 화면 좌측 상단의 쉬운모드 버튼을 누른다.
  2. 큰 UI 적용글자와 버튼 크기가 커지고 정보 밀도가 낮아진다.
  3. 단순 그리드사이즈 선택은 2x2, 맛 선택은 3x3 기준으로 보여준다.
  4. 한 주문 단위 처리여러 주문을 동시에 다루지 않고 하나씩 처리한다.
  5. 일반 결제 흐름맛 선택 후 포인트/할인, QR 결제 흐름은 동일하게 진행된다.

중간 예외사항

  • 일반모드로 돌아갈 때 기존 선택 유지 여부를 확인한다.
  • 뒤로가기, 홈, 직원 호출 버튼은 항상 크게 유지한다.
U-04

포인트/할인 결제

회원 등급, 보유 포인트, 쿠폰을 결제 금액에 반영한다.

  1. 회원 조회휴대폰 번호를 입력해 해피포인트 회원을 조회한다.
  2. 등급 확인Friend 3%, Family 5%, VIP 8% 적립률을 적용한다.
  3. 포인트 사용100원 단위로 포인트를 사용하고 최대 사용 금액을 제한한다.
  4. 할인 적용제휴 할인, 쿠폰, 이벤트 할인 정책을 반영한다.
  5. 최종 금액 계산할인과 포인트를 반영해 QR 결제 금액을 확정한다.

중간 예외사항

  • 회원 정보가 없으면 비회원 결제를 제공한다.
  • 포인트가 부족하거나 100원 미만이면 사용을 제한한다.
  • 중복 할인 불가 정책이면 하나만 선택하게 한다.
Branch Scenario

지점 시나리오

지점은 고객 주문을 실제 운영으로 처리합니다. 주문 처리, 재고 차감, 재고 신청, 추천 메뉴 관리가 중심입니다.

B-01

지점 하루 운영

주문을 오래된 순으로 처리하고 완료 알림과 재고 차감을 수행한다.

  1. 대시보드 확인오늘 주문, 매출, 품절 임박, 배송 중 신청을 확인한다.
  2. 주문 목록 정렬오래된 주문부터 제조할 수 있도록 목록을 정렬한다.
  3. 제조 완료 처리주문 완료 버튼을 누르고 고객에게 완료 알림을 보낸다.
  4. 재고 자동 차감판매된 상품 수량만큼 지점 재고가 감소한다.
  5. 바빠요 설정주문이 몰리면 예상 소요시간을 키오스크 메인에 표시한다.

중간 예외사항

  • 주문 취소 시 취소 사유를 직접 입력한다.
  • 알림 전송 실패 시 주문 상태는 유지하고 실패 로그를 남긴다.
  • 재고 차감 오류가 발생하면 완료 처리를 막는다.
B-02

재고 신청

안전재고 이하 상품을 본점에 신청하고 배송 수령으로 재고를 회복한다.

  1. 재고 확인재고가 적은 순, 카테고리별, 상품명 검색으로 부족 상품을 찾는다.
  2. 신청 상품 선택판매량이 많고 재고가 적은 상품을 우선 신청한다.
  3. 수량/사유 입력신청 수량과 프로모션 대비 등 신청 사유를 입력한다.
  4. 본점 처리 대기신청은 대기중 상태가 되고 승인/반려 결과를 기다린다.
  5. 수령 확인배송 도착 후 수령 확인을 누르면 재고가 자동 증가한다.

중간 예외사항

  • 신청 수량이나 사유가 없으면 제출할 수 없다.
  • 이미 대기중 신청이 있으면 중복 신청 경고를 보여준다.
  • 실제 수령 수량이 다르면 재고 조정 요청으로 연결한다.
B-03

상품/이벤트 운영

키오스크 노출, 추천 메뉴, 지점별 할인 맛을 관리한다.

  1. 상품 상태 확인품절, 품절 임박, 단종 상태를 확인한다.
  2. 노출 조정재고가 없는 상품을 키오스크에서 숨긴다.
  3. 표시 순서 관리지점 상황에 맞게 상품 표시 순서를 조정한다.
  4. 추천 메뉴 선택지점 추천 메뉴 2개를 선택한다.
  5. 할인 맛 선택본점 금액 할인 이벤트에 적용할 맛을 지점이 선택한다.

중간 예외사항

  • 단종 상품은 지점에서 다시 노출할 수 없다.
  • 비활성화된 이벤트는 지점에서 적용할 수 없다.
  • 추천 메뉴가 품절되면 추천 영역에서 제외한다.
B-04

지점 통계 활용

판매량과 시간대 데이터를 운영 판단에 사용한다.

  1. 기간 선택년, 월, 일 기준으로 조회 기간을 선택한다.
  2. 맛별 판매량잘 팔리는 맛을 추천 메뉴 후보로 활용한다.
  3. 사이즈별 판매량인기 사이즈와 이벤트 적용 후보를 파악한다.
  4. 시간대별 주문량혼잡 시간대를 확인해 바빠요 안내와 인력 배치에 활용한다.
  5. 재고 신청 판단잘 팔리면서 재고가 적은 맛을 우선 신청한다.

중간 예외사항

  • 판매 데이터가 없으면 빈 그래프와 안내 문구를 보여준다.
  • 취소/환불 건은 매출에서 제외하거나 별도 표시한다.
  • 품절로 못 판 상품은 인기 없는 상품과 구분한다.
Headquarters Scenario

본점 시나리오

본점은 재고 신청, 배송, 상품 정책, 이벤트와 통계를 관리하며 운영 방향을 결정합니다.

H-01

재고 신청 처리

지점 신청을 승인/반려하고 승인 건을 배송 흐름으로 넘긴다.

  1. 신청 목록 확인대기중 신청을 지점, 상품, 수량, 사유와 함께 확인한다.
  2. 상세 검토지점 현재 재고와 최근 판매량을 참고해 판단한다.
  3. 승인/반려승인 시 배송 준비로, 반려 시 반려 사유를 입력한다.
  4. 배송 연결승인된 신청은 배송 관리 화면에서 상태를 관리한다.
  5. 지점 수령 대기배송 완료는 지점이 실제 수령 확인을 해야 완료된다.

중간 예외사항

  • 반려 사유가 없으면 반려 처리할 수 없다.
  • 이미 처리된 신청은 다시 승인/반려할 수 없다.
  • 배송 정보가 없으면 배송중 상태로 변경할 수 없다.
H-02

배송 관리

배송 상태를 접수, 상품 준비, 배송 중, 배송 완료로 관리한다.

  1. 배송 대상 확인승인된 재고 신청 목록을 확인한다.
  2. 배송 정보 입력운송장번호, 담당자, 예상 도착일을 등록한다.
  3. 상태 변경접수에서 상품 준비, 배송 중으로 변경한다.
  4. 수령 확인지점이 물품을 받으면 배송 완료가 된다.
  5. 재고 반영신청 수량만큼 지점 재고가 자동 증가한다.

중간 예외사항

  • 예상 도착일이 지나면 지연 배송 목록에 표시한다.
  • 수령 수량이 다르면 재고 조정 요청으로 연결한다.
  • 배송 중 상태에서 반려로 되돌리는 것은 제한한다.
H-03

신규 지점 등록

본점은 가입 URL을 발급하고 지점은 직접 정보를 입력한다.

  1. 가입 URL 발급본점이 신규 지점장의 이메일 또는 전화번호를 입력한다.
  2. URL 전송가입 링크를 문자, 이메일, Firebase 등으로 보낸다.
  3. 지점 정보 입력지점장이 ID, 비밀번호, 지점명, 연락처, 주소를 입력한다.
  4. 분점 목록 등록가입 완료 후 본점 분점 목록에 등록된다.
  5. 운영 상태 승인필요 시 본점이 운영상태를 정상으로 변경한다.

중간 예외사항

  • 가입 URL이 만료되면 다시 발급한다.
  • ID, 이메일, 전화번호 중복을 확인한다.
  • 운영 대기 지점은 키오스크 운영을 시작할 수 없다.
H-04

상품/이벤트/쿠폰 운영

신제품, 단종, 이달의 맛, 할인, 쿠폰 발송을 관리한다.

  1. 신제품 등록상품명, 카테고리, 가격, 이미지, 출시일을 입력한다.
  2. 단종 관리단종 상품은 지점/사용자 화면에서 비노출한다.
  3. 이달의 맛 이벤트할인율과 500원 추가 사이즈업 조건을 설정한다.
  4. 금액 할인 이벤트본점이 할인액을 만들고 지점이 적용 맛을 선택한다.
  5. QR 쿠폰 발송고객 전화번호 DB를 기반으로 쿠폰을 보낸다.

중간 예외사항

  • 필수 상품 정보가 없으면 신제품을 등록할 수 없다.
  • 이벤트 기간 오류와 중복 적용 정책을 확인한다.
  • 전화번호 오류는 쿠폰 발송 실패 목록에 남긴다.
H-05

통계와 의사결정

본점은 누적 데이터를 보고 상품, 이벤트, 지점 관리 정책을 결정한다.

  1. 조회 조건 선택기간, 지점, 상품 기준을 선택한다.
  2. 상품 신청 건수 확인자주 신청되는 상품을 본점 재고 확보 우선순위로 판단한다.
  3. 지점별 판매액 확인판매액이 낮은 지점은 관리 대상, 높은 지점은 이벤트 후보로 본다.
  4. 상품별 판매액 확인인기 상품은 이벤트 후보, 저조 상품은 단종 후보로 분류한다.
  5. 리포트 다운로드필요 시 조건별 엑셀 리포트를 내려받는다.

중간 예외사항

  • 통계 데이터가 없으면 빈 상태 화면을 보여준다.
  • 취소/환불 주문은 별도 기준으로 처리한다.
  • 품절 기간이 긴 상품은 단순 판매 저조로 판단하지 않는다.
