📺 1단계: 광고 화면 진입

  • 광고 노출 대기: 사용자가 화면을 만지기 전까지 슬라이드 형태의 프로모션 광고가 재생됩니다.
  • 즉시 주문 시작: 화면 어디든 터치(클릭)하는 즉시 광고가 닫히며 주문 프로세스가 시작됩니다.
  • UI 공통 제공: 주문 전 과정에서 언제든 메인으로 돌아갈 수 있는 홈버튼(좌상단)과 직원 호출버튼(우상단)이 지원됩니다.

🛍️ 2단계: 매장/픽업 선택 분기

  • 팝업 형태의 UI: 사용자가 인지하기 쉽도록 선택창이 화면 중앙에 팝업 형태로 나타납니다.
  • 다국어 보조 지원: 외국인 사용자도 쉽게 파악할 수 있게 각 선택 버튼에 영어 서브타이틀이 병기됩니다.
  • 동선 최적화: 매장 취식을 선택하면 향후 픽업 일시 등을 묻는 단계를 지능적으로 건너뛰게 됩니다.

🌐 3단계: 언어 및 쉬운 모드 설정

  • 다국어 기능: 좌상단 버튼을 통해 한국어, 영어, 일본어, 중국어 등 4개 국어로 전체 UI를 변경할 수 있습니다.
  • 시니어/저시력자 배려: 쉬운 모드 진입 시 전체 폰트가 대폭 확대됩니다.
  • UI 직관성 극대화: 쉬운 모드에서는 아이스크림 크기를 2x2, 맛 종류를 3x3 바둑판 배열로 단순화하며, 단일 주문만 가능하게 강제합니다.

🍦 4단계: 맞춤형 메뉴 선택

  • 카테고리 우선 노출: '신제품(최근 출시 순)'과 '추천메뉴(본점 TOP2 + 지점 지정 2종)'가 기본 탭으로 제공됩니다.
  • 대용량 아이스크림 분기: 패밀리, 하프갤런 등 대용량 선택 시 불필요한 '상품 종류' 단계를 묶음 생략하고 즉시 맛 선택 화면으로 직행합니다.
  • 상태 피드백: 사용자가 선택할 때마다 화면 하단에 현재 컵의 아이스크림 용량(진행률)이 시각적으로 표시됩니다.

🛒 5단계: 부가 옵션 및 장바구니 제어

  • 대용량 특화 옵션: 숟가락 개수를 직접 입력하며, 드라이아이스 포장 시간을 설정할 수 있습니다(시간당 1개 지급, 최대 3개 한도).
  • 삭제 재확인 방어: 장바구니에서 실수로 X를 눌러 삭제되지 않도록 재확인 팝업 로직을 추가했습니다.
  • 수정 편의성: 장바구니 목록의 메뉴 아이콘을 클릭하여 이전 단계로 돌아가지 않고도 옵션을 즉시 수정합니다.

💳 6단계: 포인트 적립 및 모바일 QR 결제

  • 등급별 차등 적립: 휴대폰 번호 입력 시 등급에 따라 Friend 3%, Family 5%, VIP 8% 포인트가 적립됩니다.
  • 포인트 사용: 100원 단위 전용 키패드(+100, -100, 최대사용 버튼)를 통해 잔여 포인트를 적용합니다.
  • 결제 안정성: 결제창에 팝업되는 QR코드를 모바일로 스캔해 결제하며, 통신 실패 시 QR이 즉시 재생성됩니다. 결제 완료 시 포인트가 미적립된 상태라면 알림 팝업이 발생합니다.

🏪 지점/분점 (Branch) Flow - 매장 관리자

본점의 승인으로 시스템에 합류하며, 실시간 매장 대응과 자동화된 재고/발주를 수행

📦 주문 및 현장 대응

  • 계정 접속: 본점 발급 URL로 지점이 직접 가입 완료. 비밀번호 재설정 시 Firebase 인증
  • 주문 관리: 오래된 순 정렬, 지점이 직접 사유를 입력하여 주문 취소 가능, 완료 시 고객에게 앱 알림 발송
  • '바빠요' 모드: 주문 지연 10분 이상 시, 키오스크에 예상 대기시간(10/20/30분) 강제 노출
  • 고객 지원: 키오스크 직원 호출 시, 관리자에게 푸시 및 소리 알림 수신

📊 스마트 자동화 재고/통계

  • 재고 자동차단: 상품 재고 0개 시 키오스크 비노출 자동화. 오차 발생 시 본점에 수동 조정 요청
  • 발주 프로세스: 부족 재고 본점 발주 ➔ 배송 완료 후 지점이 수령 확인 시 매장 재고 자동 증가
  • 이벤트 유연성: 본점이 기획한 금액 할인 이벤트에 적용할 상세 '맛'을 지점이 직접 선택 적용

🏢 본점 (HQ) Flow - 최고 관리자

전 지점을 총괄하며 물류 승인, 이벤트 기획, 계정 권한 분리 등 마스터 통제 수행

🔄 권한 제어 및 물류 심사

  • 계정 및 접근: 지점 가입 URL 발송. 슈퍼관리자와 본점관리자 권한 분리 (마스터 설정 수정 제한)
  • 운영 통제: 분점의 운영 상태(정상/대기/중단)를 사유 입력과 함께 제어
  • 물류 심사: 분점 발주에 대한 승인 및 반려(사유 필수 입력). 배송 지연 건 전수 모니터링
  • 수동 조정: 지점의 실사 오차 요청 시 본점이 재고 수량을 수동으로 직접 조정

🎉 프로모션 고도화 및 커뮤니케이션

  • 이달의 맛 이벤트: 전용 할인율 지정 및 500원 추가 결제 시 사이즈업 로직 등 세밀한 기획
  • 타겟 마케팅: Firebase 기반으로 DB 고객들에게 QR코드 할인 쿠폰 일괄 발송
  • 데이터 분석: 결제수단/할인 적용 통계 및 엑셀 다운로드 제공
  • 소통 창구: 전 지점 대상 이미지 첨부 공지사항 배포 및 N:1 채팅 상담 인프라

🤝 본점(HQ)과 지점(분점)의 관계 구조

중앙 통제 시스템 기반에서 일부 현장 권한을 유연하게 위임하는 프랜차이즈 모델

🏢 본점 (최고 관리)

시스템 통제 & 물류 심사

  • • 분점(지점) 가입 승인 및 운영상태 강제 제어
  • • 전체 메뉴/이달의 맛, 사이즈업 행사 기획
  • • 분점 발주 요청의 승인/반려 권한 독점
  • • 재고 실사 불일치 시 최종 조정 권한 보유

🏪 분점/지점 (매장 운영)

현장 운영 & 예외 대응

  • • 실시간 현장 판단으로 주문 취소 및 지연 알림
  • • 금액 할인 이벤트 시 로컬 수요에 맞춘 맛 선택
  • • 배송 도착 시 직접 수령 확인(배송완료) 처리
  • • 부족 재고 파악 후 자동 알림에 따른 자가 발주

🚨 1. 알림 및 발주 신청 (지점)

특정 인기 아이스크림 재고가 안전재고 미만으로 떨어져 시스템 경고가 울립니다. 지점장은 이번 주말 예정된 '사이즈업' 이벤트를 대비해 넉넉한 수량을 입력하고, "주말 프로모션 대비 사전 확보"라는 사유와 함께 본점에 발주를 신청합니다.

🏢 2. 심사 및 배송 처리 (본점)

본점 관리자는 해당 지점의 발주 요청 건과 사유를 심사한 뒤 '승인' 처리합니다. 이후 물류창고에 지시하여 전산상 배송 상태를 순차적으로 '접수 ➔ 상품 준비 ➔ 배송 중'으로 업데이트합니다.

🔄 3. 수령 확인 및 자동 동기화 (지점)

다음 날, 배송 트럭이 지점에 도착합니다. 지점장이 물품을 확인하고 시스템에서 '배송 완료(수령 확인)' 버튼을 누르는 즉시, 품절 처리되어 키오스크에서 사라졌던 메뉴판이 부활하며 재고가 자동 반영됩니다.

🛡️ 공통 인프라 및 보안 체계

역할과 플랫폼에 구애받지 않는 강력한 기반 시스템 (S-087 ~ S-095)

보안 시스템 (Security)

  • 비밀번호 암호화: 민감 정보 유출 방지
  • 역할 기반 접근제어(RBAC): 슈퍼관리자/본점/지점간 엄격한 메뉴 권한 분리
  • SQL Injection 방지: 데이터베이스 악의적 공격 원천 차단

운영/로그 시스템 (System)

  • 결제 위변조 방지: QR결제 및 포인트 조작 방지 검증 로직
  • 감사 로그 (Audit Log): 중요 상태 변경자 및 시각 이력 기록 (재고/배송 등)
  • 예외 처리/오류 표준화: 통합된 방식으로 안정적인 서비스 응답 제공
