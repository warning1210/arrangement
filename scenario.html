베스킨라빈스 키오스크 주문 시나리오

원래모드와 쉬운모드의 주문 흐름 비교 화면

원래모드

여러 개의 메뉴를 먼저 선택한 뒤 주문 내역을 한 번에 확인하는 방식

1광고화면
광고가 슬라이드 형태로 표시된다. 사용자가 화면을 터치하면 다음 단계로 이동한다.
↓
2주문방법 선택
매장 식사 또는 픽업 방식을 선택한다.
↓
3메인화면에서 메뉴 선택
3개를 주문하는 경우, 원하는 메뉴를 메인화면에서 총 3번 클릭한다.
예시 흐름
메뉴 1 선택 → 메뉴 2 선택 → 메뉴 3 선택
↓
4컵 / 콘 / 와플콘 선택
예를 들어 싱글레귤러를 선택했다면 제공 방식을 선택한다.
컵콘와플콘
↓
5맛 선택
선택한 메뉴에 들어갈 아이스크림 맛을 고른다.
↓
6주문 내역 한 번에 확인
선택한 3개의 메뉴를 한 화면에서 한 번에 확인한다.
↓
7할인 / 적립 페이지
쿠폰, 포인트 사용, 포인트 적립 여부를 선택한다.
↓
8결제 페이지
QR 결제 또는 선택한 결제 방식으로 결제를 진행한다.
↓
9영수증 선택
영수증 출력 또는 미출력을 선택한다.
출력미출력
↓
10주문번호 출력
결제가 완료되면 주문번호가 화면에 표시된다.

쉬운모드

하나의 메뉴를 끝까지 선택한 뒤 다시 메인으로 돌아오는 방식

1광고화면
광고가 표시된다. 사용자가 화면을 터치하면 주문 화면으로 이동한다.
↓
2주문방법 선택
매장 식사 또는 픽업 방식을 선택한다.
↓
3쉬운모드 클릭
메인화면 좌측 상단의 언어 선택 버튼 옆에 있는 쉬운모드 버튼을 누른다.
UI 변화: 글자 크기 확대 / 버튼 크기 확대 / 화면 구성 단순화
↓
4쉬운모드 메인화면
메뉴 화면이 2x2 구조로 바뀌어 더 크게 표시된다.
메뉴 1메뉴 2 메뉴 3메뉴 4
↓
5메뉴 1개 선택
사용자는 한 번에 하나의 메뉴만 선택한다.
↓
6컵 / 콘 / 와플콘 선택
싱글레귤러처럼 제공 방식이 필요한 메뉴라면 컵, 콘, 와플콘 중 하나를 선택한다.
컵콘와플콘
↓
7맛 선택
맛 선택 화면은 3x3 구조로 바뀌어 크게 표시된다.
맛 1맛 2맛 3 맛 4맛 5맛 6 맛 7맛 8맛 9
↓
8메인화면으로 돌아가기
하나의 메뉴 선택이 끝나면 다음 메뉴를 고르기 위해 다시 메인화면으로 돌아간다.
3개 주문 예시
메뉴 1 선택 완료 → 메인으로 이동
메뉴 2 선택 완료 → 메인으로 이동
메뉴 3 선택 완료 → 주문 진행
↓
9할인 / 적립 페이지
모든 메뉴 선택이 끝난 후 쿠폰, 포인트 사용, 포인트 적립 여부를 선택한다.
↓
10결제 페이지
QR 결제 또는 선택한 결제 방식으로 결제를 진행한다.
↓
11영수증 선택
영수증 출력 또는 미출력을 선택한다.
출력미출력
↓
12주문번호 출력
결제가 완료되면 주문번호가 화면에 크게 표시된다.

원래모드는 여러 메뉴를 먼저 담고 한 번에 확인하는 흐름이고, 쉬운모드는 메뉴를 하나씩 완료하며 진행하는 흐름이다.

📦 지점 ↔ 본점 시나리오: 재고 긴급 발주 및 자동화 프로세스

시나리오: 주말 프로모션을 앞두고 인기 메뉴 재고가 소진될 위기에 처한 지점의 긴급 발주 흐름

sequenceDiagram autonumber participant K as 지점 키오스크 participant B as 지점 관리자 participant H as 본점 관리자 K->>B: 안전재고 미달 자동 알림 Note over B: "주말 프로모션 대비
사전 물량 확보 필요" B->>H: 부족 재고 긴급 발주 신청
(사유 입력 필수) H-->>H: 재고 여력 및 사유 심사 alt 승인 (물량 확보) H->>B: 발주 승인 및 상태 변경
(상품준비 ➔ 배송중) Note over H, B: 🚚 물리적 물품 배송 진행 B->>B: 물품 도착 박스 확인 후
시스템 '수령 확인' 버튼 클릭 B->>K: 키오스크 전산망 자동 동기화
(재고 증가 및 판매 재개) else 반려 (물량 부족/사유 불충분) H->>B: 발주 반려 (반려 사유 필수 입력)
(예: 본점 창고 일시 품절) Note right of B: 반려 사유 확인 후
대체 상품으로 발주 재신청 end

🚨 1. 알림 및 발주 신청 (지점)

특정 인기 아이스크림 재고가 안전재고 미만으로 떨어져 시스템 경고가 울립니다. 지점장은 이번 주말 예정된 '사이즈업' 이벤트를 대비해 넉넉한 수량을 입력하고, "주말 프로모션 대비 사전 확보"라는 사유와 함께 본점에 발주를 신청합니다.

🏢 2. 발주 심사 및 처리 (승인/반려)

본점은 지점의 발주 사유와 재고 여력을 심사합니다.
✔️ 승인 시: 배송 상태를(접수➔준비➔배송중) 업데이트하고 물류를 출발시킵니다.
❌ 반려 시: 본점 재고 부족 등 사유가 있을 경우 반려 사유를 필수 입력하여 반려 처리하며, 지점은 이를 확인 후 대체 재고를 재신청해야 합니다.

🔄 3. 수령 확인 및 자동 동기화 (승인 건)

배송이 승인되어 지점에 도착한 경우, 지점장이 물품을 확인하고 시스템에서 '배송 완료(수령 확인)' 버튼을 누릅니다. 클릭 즉시 품절 처리되었던 메뉴판이 부활하며 키오스크에 재고가 자동 반영됩니다.

🎉 지점 ↔ 본점 시나리오: 이벤트 기획 및 적용 프로세스

시나리오: 본점이 전국 단위 이벤트를 하달하고, 지점이 로컬 상권 맞춤형 이벤트를 역제안하는 양방향 프로모션 흐름

sequenceDiagram autonumber participant H as 본점 관리자 participant B as 지점 관리자 participant K as 지점 키오스크 Note over H, B: 1. 본점 주도 이벤트 (Top-Down) H->>B: ① 이달의 맛 할인 이벤트 배포 H->>B: ② '500원 추가 시 사이즈업' 로직 배포 B->>B: 로컬 재고 상황에 맞춰 적용할 '맛' 선택 B->>K: 키오스크 메인 배너 및 할인 즉시 적용 Note over B, H: 2. 지점 주도 이벤트 (Bottom-Up) B->>H: ① 특정 메뉴 N원 할인 신청 (사유 기재) B->>H: ② 로컬 프로모션용 사이즈업 신청 H-->>H: 마진율 및 타당성 심사 H->>B: 이벤트 최종 승인 B->>K: 키오스크에 지점 전용 이벤트 활성화

🏢 1. 본점 주도 이벤트 (Top-Down)

본점이 전국 단위로 '이달의 맛 할인'과 '500원 추가 시 사이즈업' 이벤트를 기획하여 일괄 배포합니다. 지점은 해당 이벤트 틀 안에서 본인 매장의 재고 상황에 맞춰 구체적인 적용 '맛'을 선택하여 키오스크에 노출시킵니다.

🏪 2. 지점 주도 이벤트 (Bottom-Up)

지점장이 상권 특성을 고려해 자체적인 '특정 메뉴 N원 할인'과 '로컬 사이즈업' 이벤트를 본점에 신청합니다. 이벤트 사유와 함께 상신하면, 본점은 지점의 마진 구조와 타당성을 심사한 후 최종 승인합니다.

🔄 3. 동기화 및 즉시 적용

본점 주도 이벤트의 세부 설정이 완료되거나, 지점 주도 이벤트가 승인되는 즉시 해당 지점의 키오스크 DB와 동기화됩니다. 고객 주문 시 이벤트 팝업이 노출되며 결제 화면에 할인 로직이 자동으로 반영됩니다.
